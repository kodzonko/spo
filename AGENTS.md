# Instructions for Agents

- When new files are added or autogenerated in this repository consider if they should be commited. If not extend repo's .gitignore
- After making signifficant changes to the codebase, run the tests to ensure that everything is working as expected.
- Cover new code with tests, always prefer e2e and integration tests over heavily mocked unit tests. Add unit tests as a supplementary measure when e2e and integration tests are not sufficient to cover all edge cases or recreate specific scenarios that are hard to achieve.
- As a final step of code changes run linter, type checker and formatter and try to satisfy them all. In special cases when satisfying them would result in worse code quality add inline comments to disable them for the specific code block and explain why.
- python virtualenv, dependencies and running commands should be handled by `uv`.
- Always choose latest stable library versions when adding new dependencies, unless there is a specific reason not to.
- Clearly note down all most important A vs B decisions we agree on (if possible with reasoning behind them) while planning and implementing in clearly named existing or new markdown files under docs/ directory. This will help us keep track of our design decisions and the reasoning behind them.
